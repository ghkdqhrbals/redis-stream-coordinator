# Redis Stream Coordinator 0.2.0 릴리즈

릴리즈 날짜: 2026-06-08

이 minor 릴리즈는 공개 client surface를 Spring Boot/Kotlin 모듈에 집중하고, producer API와 샘플을 named `StreamProducer` bean 중심으로 정리한다. 실험적 Python client는 active release surface에서 제거했다.

## 릴리즈 모듈

| Module | Artifact | Runtime | Status | Scope |
| --- | --- | --- | --- | --- |
| Coordinator server | Docker image | Spring Boot/Kotlin | GHCR로 릴리즈 | Coordinator API, heartbeat control plane, Redis metadata store, monitoring, ACL, audit, console/API docs |
| Spring Boot starter | `io.github.ghkdqhrbals:redisstream-spring-boot-starter:0.2.0` | Spring Boot/Kotlin | Maven Central 릴리즈 | Consumer heartbeat lifecycle, `@StreamListener`, Redis Stream polling adapter, named `StreamProducer` beans, producer routing, publishing |
| Shared protocol core | `io.github.ghkdqhrbals:redisstream-core:0.2.0` | Kotlin/JVM | Maven Central 릴리즈 | Coordination version contract, timing defaults, compatibility metadata |
| Python client | N/A | Python | 제거 | 0.2.0에서는 Python package를 릴리즈하지 않는다 |

Redis command 참고:

* Producer는 scale-in으로 retired 된 shard를 다시 만들지 않기 위해 `XADD NOMKSTREAM`을 사용한다.
* Consumer ACK는 기본적으로 `XACK`을 사용할 수 있다.
* `XACKDEL`은 Redis 8.2+가 필요하다.
* `XNACK`은 Redis 8.8+가 필요하다.

## Producer API

애플리케이션은 logical stream마다 하나의 `StreamProducer` bean을 정의하고, 필요한 producer를 bean name 또는 `@Qualifier`로 주입해야 한다.

```kotlin
@Bean("ordersStreamProducer")
fun ordersStreamProducer(
    coordinatorClient: CoordinatorClient,
    redisConnectionFactory: RedisConnectionFactory,
): StreamProducer =
    StreamProducer(
        streamPrefix = "orders",
        consumerGroupName = "orders-consumer",
        client = coordinatorClient,
        redisConnectionFactory = redisConnectionFactory,
        routingRefreshInterval = Duration.ofSeconds(30),
        xadd = RedisStreamXAddConfiguration(maxLen = 10_000_000),
    )
```

```kotlin
class OrderPublisher(
    @Qualifier("ordersStreamProducer")
    private val ordersProducer: StreamProducer,
) {
    fun publish(orderId: String, payload: String) {
        ordersProducer.publish(orderId, mapOf("payload" to payload))
    }
}
```

이 구조는 하나의 애플리케이션이 여러 stream prefix로 publish할 때 `ProducerRoutingProperties` 타입 주입이 애매해지는 문제를 피한다.

## Maven Central 사용

```kotlin
implementation("io.github.ghkdqhrbals:redisstream-spring-boot-starter:0.2.0")
```

Maven `groupId`는 GitHub 계정으로 namespace 검증이 쉬운 `io.github.ghkdqhrbals`를 사용한다. Kotlin package namespace는 계속 `com.redisstream`으로 유지한다.

## Python Client 제거

Repository는 더 이상 `clients/python`, `redisstream-coordinator`, Python client CI workflow를 제공하지 않는다. Python 지원은 추후 별도 package로 다시 검토할 수 있지만, 0.2.0은 JVM/Spring Boot 릴리즈이다.
