# Redis Stream Coordinator 0.1.0 릴리즈

릴리즈 날짜: 2026-06-08

이 릴리즈는 Redis Stream Coordinator의 첫 공식 모듈 세트이다. Spring Boot/Kotlin 기반 coordinator 및 starter 모듈과 sync-first Python producer/consumer client를 포함한다.

## 릴리즈 모듈

| 모듈 | Artifact/package | Runtime | 상태 | 제공 기능 |
| --- | --- | --- | --- | --- |
| Coordinator server | `coordinator-server` Docker/JAR | Spring Boot/Kotlin | 릴리즈 | Coordinator API, group metadata, member heartbeat, assignment, resharding, Redis-backed state, ACL/audit, monitoring, Scalar API docs |
| Spring Boot starter | `com.redisstream:redisstream-spring-boot-starter:0.1.0` | Spring Boot/Kotlin | 릴리즈 | Consumer heartbeat lifecycle, `@StreamListener`, Redis Stream polling adapter, producer routing cache, publisher |
| Shared protocol core | `com.redisstream:redisstream-core:0.1.0` | Kotlin/JVM | 릴리즈 | Coordination version contract, timing defaults, compatibility metadata |
| Python client | `redisstream-coordinator==0.1.0` | Python | 릴리즈 | Sync producer, sync consumer, heartbeat client, Redis Stream command template |

## 설치

Spring Boot starter:

```kotlin
implementation("com.redisstream:redisstream-spring-boot-starter:0.1.0")
```

PyPI 배포 전에는 Git repository에서 설치한다.

```bash
pip install "git+https://github.com/ghkdqhrbals/redis-stream-coordinator.git@v0.1.0#subdirectory=clients/python"
```

PyPI 배포가 설정된 뒤:

```bash
pip install redisstream-coordinator==0.1.0
```

## 테스트 게이트

릴리즈 전 모듈별 테스트를 분리해서 수행한다.

| 게이트 | 명령 |
| --- | --- |
| Coordinator server | `./gradlew :coordinator-server:test` |
| Spring Boot starter/core | `./gradlew :redisstream-core:test :redisstream-spring-boot-starter:test` |
| Python unit tests | `python -m unittest discover -s clients/python/tests` |
| Python Redis integration | `RSC_REAL_REDIS_URL=redis://127.0.0.1:6379 python -m unittest clients/python/tests/test_integration_real_redis.py` |

Python Redis integration은 실제 Redis container와 in-process coordinator stub을 사용한다. Producer publish, heartbeat assignment, `XREADGROUP`, `XACK`, stale route retry with `XADD NOMKSTREAM`을 검증한다.

## 보장 경계

모든 릴리즈 client의 처리 모델은 at-least-once이다. Exactly-once side effect, global event-id deduplication, retry policy, DLQ policy, business idempotency는 application 책임이다.
