# Redis Stream Coordinator 0.1.0 Release

Release date: 2026-06-08

This release publishes the first coordinated module set for Redis Stream Coordinator. It includes the Spring Boot/Kotlin control plane and starter modules, plus a sync-first Python producer/consumer client.

## Released Modules

| Module | Artifact/package | Runtime | Status | Provides |
| --- | --- | --- | --- | --- |
| Coordinator server | `coordinator-server` Docker/JAR | Spring Boot/Kotlin | Released | Coordinator API, group metadata, member heartbeat, assignment, resharding, Redis-backed state, ACL/audit, monitoring, Scalar API docs |
| Spring Boot starter | `com.redisstream:redisstream-spring-boot-starter:0.1.0` | Spring Boot/Kotlin | Released | Consumer heartbeat lifecycle, `@StreamListener`, Redis Stream polling adapter, producer routing cache, publisher |
| Shared protocol core | `com.redisstream:redisstream-core:0.1.0` | Kotlin/JVM | Released | Coordination version contract, timing defaults, compatibility metadata |
| Python client | `redisstream-coordinator==0.1.0` | Python | Released | Sync producer, sync consumer, heartbeat client, Redis Stream command template |

## Install

Spring Boot starter:

```kotlin
implementation("com.redisstream:redisstream-spring-boot-starter:0.1.0")
```

Python from the repository until PyPI publishing is configured:

```bash
pip install "git+https://github.com/ghkdqhrbals/redis-stream-coordinator.git@v0.1.0#subdirectory=clients/python"
```

After PyPI publishing is configured:

```bash
pip install redisstream-coordinator==0.1.0
```

## Test Gates

The release must pass separate module test gates:

| Gate | Command |
| --- | --- |
| Coordinator server | `./gradlew :coordinator-server:test` |
| Spring Boot starter/core | `./gradlew :redisstream-core:test :redisstream-spring-boot-starter:test` |
| Python unit tests | `python -m unittest discover -s clients/python/tests` |
| Python Redis integration | `RSC_REAL_REDIS_URL=redis://127.0.0.1:6379 python -m unittest clients/python/tests/test_integration_real_redis.py` |

The Python Redis integration gate uses a real Redis container and a local in-process coordinator stub. It verifies producer publish, heartbeat assignment, `XREADGROUP`, `XACK`, and stale route retry with `XADD NOMKSTREAM`.

## Guarantee Boundary

All released clients follow at-least-once processing. Exactly-once side effects, global event-id deduplication, retry policy, DLQ policy, and business idempotency remain application responsibilities.
