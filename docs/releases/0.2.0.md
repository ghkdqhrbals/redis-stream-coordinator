# Redis Stream Coordinator 0.2.0 Release

Release date: 2026-06-08

This minor release focuses the public client surface on the Spring Boot/Kotlin modules and replaces the producer sample/API guidance with named `StreamProducer` beans. The experimental Python client has been removed from the active release surface.

## Released Modules

| Module | Artifact | Runtime | Status | Scope |
| --- | --- | --- | --- | --- |
| Coordinator server | Docker image | Spring Boot/Kotlin | Released via GHCR | Coordinator API, heartbeat control plane, Redis metadata store, monitoring, ACL, audit, and console/API docs |
| Spring Boot starter | `io.github.ghkdqhrbals:redisstream-spring-boot-starter:0.2.0` | Spring Boot/Kotlin | Released to Maven Central | Consumer heartbeat lifecycle, `@StreamListener`, Redis Stream polling adapter, named `StreamProducer` beans, producer routing, publishing |
| Shared protocol core | `io.github.ghkdqhrbals:redisstream-core:0.2.0` | Kotlin/JVM | Released to Maven Central | Coordination version contract, timing defaults, compatibility metadata |
| Python client | N/A | Python | Removed | No Python package is released in 0.2.0 |

Redis command notes:

* `XADD NOMKSTREAM` is used by producers to avoid recreating retired scale-in shards.
* `XACK` is generally available for consumer acknowledgement.
* `XACKDEL` requires Redis 8.2+.
* `XNACK` requires Redis 8.8+.

## Producer API

Applications should define one `StreamProducer` bean per logical stream and inject the required producer by bean name or `@Qualifier`.

```kotlin
@Bean("ordersStreamProducer")
fun ordersStreamProducer(
    coordinatorClient: CoordinatorClient,
    redisConnectionFactory: RedisConnectionFactory,
): StreamProducer =
    StreamProducer(
        streamPrefix = "orders",
        consumerGroupName = "orders-consumer",
        client = coordinatorClient,
        redisConnectionFactory = redisConnectionFactory,
        routingRefreshInterval = Duration.ofSeconds(30),
        xadd = RedisStreamXAddConfiguration(maxLen = 10_000_000),
    )
```

```kotlin
class OrderPublisher(
    @Qualifier("ordersStreamProducer")
    private val ordersProducer: StreamProducer,
) {
    fun publish(orderId: String, payload: String) {
        ordersProducer.publish(orderId, mapOf("payload" to payload))
    }
}
```

This avoids ambiguous `ProducerRoutingProperties` injection when an application publishes to multiple stream prefixes.

## Maven Central Usage

```kotlin
implementation("io.github.ghkdqhrbals:redisstream-spring-boot-starter:0.2.0")
```

The Maven `groupId` is `io.github.ghkdqhrbals` because the namespace can be verified through the GitHub account. Kotlin package names remain `com.redisstream`.

## Removed Python Client

The repository no longer ships `clients/python`, `redisstream-coordinator`, or the Python client CI workflow. Python support may be reconsidered later as a separate package, but 0.2.0 is a JVM/Spring Boot release.
