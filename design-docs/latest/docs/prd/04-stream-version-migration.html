Stream Version Migration and Routing

Routing Contract

мқҙ л¬ём„ңм—җм„ң topic partitionм—җ н•ҙлӢ№н•ҳлҠ” мөңмҶҢ routing лӢЁмң„лҠ” Redis Stream shardмқҙлӢӨ.

streamVersion, activeWriteVersion, readableVersions, fromVersion, toVersionмқҖ лӘЁл‘җ м •мҲҳмқҙлӢӨ. v1, v2 к°ҷмқҖ prefix л¬ёмһҗм—ҙмқ„ м ҖмһҘн•ҳм§Җ м•ҠлҠ”лӢӨ.

ProducerлҠ” local shard countлҘј м§Ғм ‘ м“°м§Җ м•ҠлҠ”лӢӨ.

ProducerлҠ” coordinatorмқҳ producer routing metadata endpointлҘј мәҗмӢңн•ңлӢӨ.

GET /coord/v1/streams/{streamPrefix}/groups/{consumerGroup}/producer-routing

metadata = metadataCache.activeWrite(streamPrefix, consumerGroup)
shardIndex = routeV1(partitionKey, metadata.shardCount)
streamKey = format(metadata.streamKeyPattern, metadata.activeWriteVersion, shardIndex)

metadataVersionмқҙ л°”лҖҢл©ҙ producerлҠ” cached routing metadataлҘј к°ұмӢ н•ҙм•ј н•ңлӢӨ.

Routing is a v1 protocol contract, not group metadata. The producer computes a 32-bit Murmur3 hash, rejects values in the incomplete modulo tail, and rehashes deterministically until the value maps uniformly into [0, shardCount). Future incompatible routing changes must use a new protocol/API version instead of storing per-group hash settings.

Routing and Duplicate Delivery Constraints

Routing determinism is scoped to one routing metadata snapshot:

{producerRoutingProtocolVersion, activeWriteVersion, shardCount, partitionKey}

лҸҷмқј partition keyлқјлҸ„ shard scale-out/inмңјлЎң activeWriteVersion лҳҗлҠ” shardCountк°Җ л°”лҖҢл©ҙ лӢӨлҘё Redis Stream shardлЎң routeлҗ  мҲҳ мһҲлӢӨ. мҳҲлҘј л“Өм–ҙ order-123мқҙ version 1, shard count 4м—җм„ңлҠ” shard 2лЎң routeлҗҳлҚ”лқјлҸ„, scale-out нӣ„ version 2, shard count 8м—җм„ңлҠ” shard 6мңјлЎң routeлҗ  мҲҳ мһҲлӢӨ.

Producer retryмҷҖ shard scale-out/inмқҖ мӨ‘ліө publish мӢңлҸ„лҘј л§Ңл“Ө мҲҳ мһҲлӢӨ. мқҙ н”„лЎңм қнҠёлҠ” м „мІҙ streamPrefix, м „мІҙ shard, м „мІҙ stream versionмқҳ лҸҷмқј eventIdлҘј м „м—ӯмңјлЎң deduplicateн•ҳм§Җ м•ҠлҠ”лӢӨ.

мӨ‘мҡ” мӢңлӮҳлҰ¬мҳӨ:

1. producer Aк°Җ eventId=E, partitionKey=order-123мқ„ old routing metadataлЎң version 1 shard 2м—җ XADDн•ңлӢӨ.
2. Redis мқ‘лӢөмқҙ producerм—җкІҢ лҸ„лӢ¬н•ҳкё° м „м—җ л„ӨнҠёмӣҢнҒ¬ мҳӨлҘҳк°Җ л°ңмғқн•ңлӢӨ.
3. мҡҙмҳҒмһҗк°Җ shard scale-outмқ„ мӢӨн–үн•ҙ activeWriteVersion=2, shardCount=8лЎң м „нҷҳн•ңлӢӨ.
4. producer A лҳҗлҠ” лӢӨлҘё producerк°Җ к°ҷмқҖ eventId=EлҘј retryн•ҳл©ҙм„ң мғҲ routing metadataлҘј мӮ¬мҡ©н•ңлӢӨ.
5. к°ҷмқҖ partitionKeyк°Җ version 2 shard 6мңјлЎң routeлҗңлӢӨ.
6. кІ°кіјм ҒмңјлЎң лҸҷмқј eventId=Eк°Җ м„ңлЎң лӢӨлҘё stream version/shardм—җ мЎҙмһ¬н•  мҲҳ мһҲлӢӨ.

л”°лқјм„ң лӢӨмқҢ мҡҙмҳҒ м ңм•Ҫмқ„ л‘”лӢӨ.

  • мӨ‘ліөм—җ лҜјк°җн•ң workloadм—җм„ңлҠ” shard scale-out/in мӨ‘ produceлҘј мҲҳн–үн•ҳм§Җ м•ҠлҠ”лӢӨ.
  • scale мҡ”мІӯ м „ producerлҘј quiesceн•ҳкі , in-flight XADDмҷҖ publish retry windowк°Җ drainлҗң л’Ө scaleмқ„ мӢӨн–үн•ңлӢӨ.
  • scale мҷ„лЈҢ нӣ„ producer routing cacheлҘј refreshн•ҳкі  мғҲ metadataлЎң produceлҘј мһ¬к°ңн•ңлӢӨ.
  • producer trafficмқ„ л©Ҳм¶ң мҲҳ м—ҶлҠ” workloadлҠ” producer semanticsлҘј at-least-onceлЎң ліҙкі , application domainмқҳ idempotency, deduplication, unique constraint, compensationмңјлЎң мӨ‘ліө side effectлҘј л°©м§Җн•ңлӢӨ.

Coordinator Admin Scale Request

Shard count ліҖкІҪмқҖ member startupмқҙлӮҳ application.yaml desired spec syncлЎң мӢңмһ‘н•ҳм§Җ м•ҠлҠ”лӢӨ. мҡҙмҳҒмһҗ лҳҗлҠ” л°°нҸ¬ мһҗлҸҷнҷ”к°Җ Coordinator Admin APIм—җ scale-out/inмқ„ мҡ”мІӯн•ҳкі , coordinatorк°Җ metadataлҘј кІҖмҰқн•ң л’Ө migrationмқ„ мӢңмһ‘н•ңлӢӨ. producer, consumer, direct Redis key writeлҠ” shard countлҘј ліҖкІҪн•  мҲҳ м—ҶлӢӨ. мӨ‘ліөм—җ лҜјк°җн•ң groupм—җм„ңлҠ” мқҙ APIк°Җ producer quiescence мқҙнӣ„м—җл§Ң нҳём¶ңлҗҳм–ҙм•ј н•ңлӢӨ.

Member startupм—җм„ң мҲҳн–үн•ҳлҠ” мқјмқҖ лӢӨмқҢмңјлЎң м ңн•ңн•ңлӢӨ.

  • coordinator metadataлҘј мқҪм–ҙ active write versionкіј readable versionsлҘј мәҗмӢңн•ңлӢӨ.
  • м§Ғм ‘ мғқм„ұн•ң memberIdлЎң heartbeatлҘј мӢңмһ‘н•ңлӢӨ.
  • heartbeatлҠ” coordinator APIмқҳ {streamPrefix, consumerGroup} pathлЎң ліҙлӮёлӢӨ.

  • local YAMLмқҳ maxConcurrencyлҠ” runtime upper boundмқј лҝҗ, coordinatorмқҳ server-side consumer concurrency policyлҘј ліҖкІҪн•ҳм§Җ м•ҠлҠ”лӢӨ.

Admin API Semantics

мҙҲкё° group мғқм„ұкіј shard scale-out/inмқҖ Coordinator Admin APIлЎңл§Ң мҲҳн–үн•ңлӢӨ.

Source of truth:

  • shard countлҠ” coordinator stream metadataк°Җ мң мқјн•ң source of truthмқҙлӢӨ.
  • consumer maxConcurrencyлҠ” coordinator consumer concurrency policyк°Җ мң мқјн•ң server-side source of truthмқҙлӢӨ.
  • maxConcurrencyлҠ” partition/shard к°ңмҲҳк°Җ м•„лӢҲлқј member лӮҙл¶Җ consumer worker мҲҳмқҙлӢӨ.
  • member heartbeatлҠ” нҳ„мһ¬ runtime м—¬мң  consumer worker мҲҳлҘј ліҙкі н•  мҲҳ мһҲм§Җл§Ң, shard countлӮҳ server-side maxConcurrencyлҘј ліҖкІҪн•ҳм§Җ лӘ»н•ңлӢӨ.

Create Group

POST /coord/v1/streams/{streamPrefix}/groups/{consumerGroup}

{
  "initialShardCount": 12,
  "versionPolicy": "AUTO_INCREMENT",
  "consumerConcurrencyPolicy": {
    "defaultMaxConcurrency": 12
  },
  "reason": "create summary group",
  "requestedBy": "deploy-automation"
}

initialShardCountмҷҖ consumerConcurrencyPolicy.defaultMaxConcurrencyк°Җ мғқлһөлҗҳл©ҙ coordinator YAMLмқҳ defaults к°’мқ„ мӮ¬мҡ©н•ңлӢӨ. мҡ”мІӯм—җ к°’мқҙ мһҲмңјл©ҙ н•ҙлӢ№ group metadataм—җ к°ңлі„ м„Өм •мңјлЎң м ҖмһҘн•ңлӢӨ.

мІҳлҰ¬ мҲңм„ң:

  1. group metadataк°Җ м—Ҷмңјл©ҙ stream version 1мқ„ мғқм„ұн•ңлӢӨ.
  2. 1 shard streamкіј Redis consumer groupмқ„ мғқм„ұн•ңлӢӨ.
  3. server-side consumer concurrency policyлҘј м ҖмһҘн•ңлӢӨ.
  4. activeWriteVersion=1, readableVersions=[1], groupEpoch=1мқ„ м ҖмһҘн•ңлӢӨ.
  5. group metadataк°Җ мқҙлҜё мһҲмңјл©ҙ 409 ConflictлЎң кұ°м Ҳн•ңлӢӨ.

Update Consumer Concurrency

Shard count ліҖкІҪ м—Ҷмқҙ member лӮҙл¶Җ consumer worker мҲҳ мғҒн•ңл§Ң л°”кҝҖ л•ҢлҠ” consumer concurrency APIлҘј мӮ¬мҡ©н•ңлӢӨ. мқҙ APIлҠ” мғҲ stream versionмқ„ л§Ңл“Өм§Җ м•Ҡкі  metadataVersionмқ„ мҰқк°ҖмӢңнӮЁ л’Ө member heartbeat responseмқҳ assignedMaxConcurrencyлҘј к°ұмӢ н•ңлӢӨ. assignment weight policyк°Җ maxConcurrencyлҘј мӮ¬мҡ©н•ҳлҠ” кІҪмҡ°м—җл§Ң groupEpochлҸ„ мҰқк°ҖмӢңнӮӨкі  target assignmentлҘј лӢӨмӢң кі„мӮ°н•ңлӢӨ.

PATCH /coord/v1/streams/{streamPrefix}/groups/{consumerGroup}/consumer-concurrency

{
  "defaultMaxConcurrency": 24,
  "memberOverrides": [
    {"memberName": "summary-consumer-a", "maxConcurrency": 16}
  ],
  "reason": "raise server-side consumer concurrency",
  "requestedBy": "deploy-automation"
}

мІҳлҰ¬ к·ңм№ҷ:

  • ліҖкІҪлҗң consumer concurrency policyлҠ” member heartbeat responseмқҳ assignedMaxConcurrencyлЎң м „нҢҢлҗңлӢӨ.
  • memberк°Җ heartbeatм—җм„ң лҚ” нҒ° runtime consumer worker мҲҳлҘј ліҙкі н•ҙлҸ„ server-side policyлҘј мҙҲкіјн•ҙм„ң workerлҘј м—ҙ мҲҳ м—ҶлӢӨ.
  • concurrency м¶•мҶҢлҠ” partition/shard мҲҳлҘј мӨ„мқҙм§Җ м•ҠлҠ”лӢӨ. memberлҠ” мҶҢмң  shardлҘј лҚ” м ҒмқҖ consumer workerлЎң multiplexingн•ңлӢӨ.
  • coordinatorлҠ” Kafka-style rebalance control planeмқҙлҜҖлЎң лі„лҸ„ global min/max concurrency configлҘј л‘җм§Җ м•ҠлҠ”лӢӨ.
  • мҡ”мІӯк°’мқҙ нҳ„мһ¬ policyмҷҖ к°ҷмңјл©ҙ metadataлҘј мғҲлЎң м“°м§Җ м•Ҡкі  нҳ„мһ¬ policyлҘј л°ҳнҷҳн•ңлӢӨ.

Migration Completion

old versionмқ„ DEPRECATEDлЎң л°”кҫёл Өл©ҙ лӢӨмқҢ мЎ°кұҙмқ„ л§ҢмЎұн•ҙм•ј н•ңлӢӨ.

  • old versionмқҳ target assignmentк°Җ л№„м–ҙ мһҲлӢӨ.
  • лӘЁл“  live member ownedShardsм—җм„ң old version shardк°Җ м ңкұ°лҗҗлӢӨ.
  • old versionмқ„ мІҳлҰ¬н•ҳлҚҳ memberк°Җ drain completeлҘј ліҙкі н–ҲлӢӨ.

Rollback Policy

Cutover м§Ғнӣ„ л¬ём ңк°Җ мһҲмңјл©ҙ coordinatorлҠ” active write pointerлҘј old versionмңјлЎң лҗҳлҸҢлҰҙ мҲҳ мһҲлӢӨ. лӢЁ, rollback window м•Ҳм—җм„ңл§Ң н—Ҳмҡ©н•ҳкі  new versionм—җ мқҙлҜё writeлҗң messageлҠ” мҡҙмҳҒ м •мұ…м—җ л”°лқј drain лҳҗлҠ” replayн•ңлӢӨ.
