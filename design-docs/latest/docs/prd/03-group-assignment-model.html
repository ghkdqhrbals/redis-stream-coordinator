Group State

stateлҠ” groupмқҙ target assignmentм—җ м–ҙлҠҗ м •лҸ„ мҲҳл ҙн–ҲлҠ”м§ҖлҘј лӮҳнғҖлӮёлӢӨ.

State Meaning
EMPTY мӮҙм•„ мһҲлҠ” memberк°Җ м—Ҷкі  readable shard ownerлҸ„ м—ҶлӢӨ.
ASSIGNING groupEpoch > assignmentEpochлқјм„ң coordinatorк°Җ мғҲ target assignmentлҘј кі„мӮ°н•ҙм•ј н•ңлӢӨ.
RECONCILING target assignmentлҠ” кі„мӮ°лҗҗм§Җл§Ң мқјл¶Җ memberк°Җ revoke/assignмқ„ м•„м§Ғ мҷ„лЈҢн•ҳм§Җ м•Ҡм•ҳлӢӨ.
STABLE лӘЁл“  active memberмқҳ current assignmentк°Җ target assignmentмҷҖ к°ҷмқҖ epochмңјлЎң мҲҳл ҙн–ҲлӢӨ.

Stream version migrationмқҳ PREPARING, ACTIVE, DRAINING, DEPRECATEDлҠ” group stateк°Җ м•„лӢҲлқј stream version stateмқҙлӢӨ.

State transition:

EMPTY
  -> ASSIGNING      first live member joins

STABLE
  -> ASSIGNING      member join, member leave, idle removal, or readable metadata change
  -> EMPTY          last member leaves and no readable shard can be owned

ASSIGNING
  -> RECONCILING    target assignment persisted for current groupEpoch

RECONCILING
  -> STABLE         all active members converge
  -> ASSIGNING      another metadata change happens before convergence

Epoch Model

epochмқҖ мӢңк°„мқҙ м•„лӢҲлқј stale actorлҘј л§үкё° мң„н•ң м„ёлҢҖ лІҲнҳёмқҙлӢӨ. coordinatorлҠ” epochмқ„ лӢЁмЎ° мҰқк°ҖмӢңнӮӨкі , memberлҠ” мһҗмӢ мқҙ л“Өкі  мһҲлҠ” epochмқҙ мөңмӢ мқём§Җ кІҖмҰқн•ң л’Ө assignmentлҘј м Ғмҡ©н•ңлӢӨ.

Epoch Owner Increments When Used For
groupEpoch coordinator member join/leave, idle removal, shard migration, readable metadata ліҖкІҪ мғҲ target assignmentк°Җ н•„мҡ”н•ңм§Җ нҢҗлӢЁ
assignmentEpoch coordinator target assignmentлҘј мғҲ group metadata кё°мӨҖмңјлЎң м ҖмһҘн•  л•Ң target assignmentк°Җ м–ҙл–Ө group epoch кё°мӨҖмқём§Җ н‘ңмӢң
memberEpoch coordinator memberк°Җ мғҲ assignmentлҘј м Ғмҡ©н•ҙм•ј н•ҳкұ°лӮҳ fencingлҗ  л•Ң stale member heartbeat/read/ack м°ЁлӢЁ

Invariant:

  • assignmentEpochмқҖ target assignmentлҘј кі„мӮ°н•ң groupEpochкіј к°ҷм•„м•ј н•ңлӢӨ.
  • active memberмқҳ memberEpochмқҖ мһҗмӢ м—җкІҢ лӮҙл Өк°„ target assignment epochмңјлЎң мҲҳл ҙн•ҙм•ј н•ңлӢӨ.
  • member data-planeмқҖ memberEpochкіј assignmentEpochмқҙ мң нҡЁн•ң assignmentм—җм„ңл§Ң read/ackн•ңлӢӨ.
  • STABLE groupм—җм„ңлҠ” active memberмқҳ current assignment н•©м§‘н•©мқҙ target assignmentмҷҖ к°ҷм•„м•ј н•ңлӢӨ.

Member Removal State

Member м ңкұ°лҠ” л‘җ кІҪлЎңлЎң м§„н–үлҗңлӢӨ.

  • graceful scale-in: memberк°Җ memberEpoch=-1 heartbeatлҘј ліҙлӮҙл©ҙ coordinatorк°Җ н•ҙлӢ№ memberмқҳ assignmentлҘј л№„мҡ°кі  revoke ackлҘј кё°лӢӨлҰ°лӢӨ.
  • idle removal: heartbeatк°Җ member-lease-ttlмқ„ л„ҳкё°л©ҙ coordinatorк°Җ memberлҘј EXPIREDлЎң fencingн•ҳкі  мһ¬н• лӢ№н•ңлӢӨ.

EXPIRED member metadataлҠ” мҰүмӢң мӮӯм ңн•ҳм§Җ м•ҠлҠ”лӢӨ. мҡҙмҳҒ л””лІ„к№…кіј group мқҙл Ҙ м¶”м Ғмқ„ мң„н•ҙ мһҘкё°к°„ мң м§Җн•ңлӢӨ.

Sticky Partition Assignment

мқҙ м„Өкі„лҠ” sticky partition assignmentлҘј кі м • м „м ңлЎң л‘”лӢӨ. memberк°Җ м„ нғқн•ҳкұ°лӮҳ heartbeatлЎң ліҙкі н•  к°’мқҙ м•„лӢҲлӢӨ.

лӘ©н‘ң:

  • shardлҘј live member мӮ¬мқҙм—җ к· л“ұ л¶„л°°н•ңлӢӨ.
  • кё°мЎҙ ownerлҘј мөңлҢҖн•ң мң м§Җн•ңлӢӨ.
  • revokeк°Җ н•„мҡ”н•ң shardл§Ң мқҙлҸҷн•ңлӢӨ.
  • ACTIVEмҷҖ DRAINING readable versionмқ„ лӘЁл‘җ assignment лҢҖмғҒмңјлЎң нҸ¬н•Ён•ңлӢӨ.

Sticky partition assignmentлҠ” нҳ„мһ¬ target assignmentлҘј мһ…л ҘмңјлЎң мӮ¬мҡ©н•ҙ movement costлҘј мөңмҶҢнҷ”н•ңлӢӨ. scale-outм—җм„ңлҠ” н•„мҡ”н•ң мқјл¶Җ shardл§Ң мӢ к·ң memberлЎң мқҙлҸҷн•ҳкі , scale-inм—җм„ңлҠ” мӮ¬лқјм§ҖлҠ” memberмқҳ shardл§Ң мӮҙм•„ мһҲлҠ” memberм—җкІҢ мһ¬л¶„л°°н•ңлӢӨ.

Sticky Assignment Algorithm

Input:

  • readable shard set: ACTIVEмҷҖ DRAINING stream versionмқҳ лӘЁл“  shard
  • live member set: ACTIVE лҳҗлҠ” STARTING member мӨ‘ member-lease-ttlмқҙ мң нҡЁн•ң member
  • excluded member set: LEAVING, EXPIRED, FENCED member
  • previous target assignment
  • optional member weight: consumer worker мҲҳлҘј load balancing к°ҖмӨ‘м№ҳлЎңл§Ң мӮ¬мҡ©н•  мҲҳ мһҲлӢӨ.

Steps:

  1. previous ownerк°Җ live memberмқҙл©ҙ кё°мЎҙ shard ownerлҘј мҡ°м„  мң м§Җн•ңлӢӨ.
  2. excluded memberк°Җ к°Җм§„ shardлҘј unassigned setмңјлЎң мҳ®кёҙлӢӨ.
  3. unassigned shardлҘј {streamVersion, shardIndex} кё°мӨҖмңјлЎң deterministic sortн•ңлӢӨ.
  4. live memberлҘј нҳ„мһ¬ shard load, optional member weight, memberId мҲңмңјлЎң м •л ¬н•ңлӢӨ.
  5. к°Ғ unassigned shardлҘј к°ҖмһҘ лҚң м°¬ memberм—җкІҢ л°°м •н•ңлӢӨ.
  6. previous ownerмҷҖ target ownerк°Җ лӢӨлҘҙл©ҙ coordinatorлҠ” previous ownerмқҳ assignment.assignedShardsм—җм„ң н•ҙлӢ№ shardлҘј м ңмҷён•ңлӢӨ.
  7. coordinatorлҠ” revoke ack лҳҗлҠ” expired member fencingмқ„ нҷ•мқён•ң л’Ө target ownerмқҳ assignment.assignedShardsм—җ н•ҙлӢ№ shardлҘј нҸ¬н•Ён•ңлӢӨ.

maxConcurrency мІҳлҰ¬:

  • maxConcurrencyлҠ” member лӮҙл¶Җ consumer worker мҲҳмқҙлӢӨ.
  • maxConcurrencyлҠ” topic partition мҲҳлӮҳ Redis Stream shard мҲҳлҘј мқҳлҜён•ҳм§Җ м•ҠлҠ”лӢӨ.
  • memberк°Җ shardлҘј м—¬лҹ¬ к°ң мҶҢмң н•  мҲҳ мһҲкі , мҶҢмң  shardлҘј assignedMaxConcurrency мқҙн•ҳмқҳ consumer workerк°Җ multiplexingн•ҙм„ң мқҪлҠ”лӢӨ.
  • coordinatorлҠ” maxConcurrencyлҘј мІҳлҰ¬лҹү к°ҖмӨ‘м№ҳлЎң м°ёкі н•  мҲҳ мһҲм§Җл§Ң, shard count ліҖкІҪкіј лҸҷмқјн•ҳкІҢ м·Ёкёүн•ҳм§Җ м•ҠлҠ”лӢӨ.
