Stream Key лӘЁлҚё

CoordinatorлҠ” stream name version tokenмқ„ л¶ҷмқҙкұ°лӮҳ кҙҖлҰ¬н•ҳм§Җ м•ҠлҠ”лӢӨ. streamPrefixлҠ” мӮ¬мҡ©мһҗк°Җ м •н•ҳлҠ” Redis Stream namespaceлӢӨ. м• н”ҢлҰ¬мјҖмқҙм…ҳмқҙ versioned nameмқ„ мӣҗн•ҳл©ҙ orders-blue, orders.blueмІҳлҹј streamPrefix м•Ҳм—җ м§Ғм ‘ нҸ¬н•Ён•ҙм•ј н•ңлӢӨ.

CoordinatorлҠ” shard indexл§Ң л¶ҷмқёлӢӨ.

streamKey = "{streamPrefix}:{shardIndex}"

мҳҲмӢң:

мӮ¬мҡ©мһҗ streamPrefix Shard count Coordinatorк°Җ мғқм„ұн•ҳлҠ” stream key
orders 3 orders:0, orders:1, orders:2
orders-blue 2 orders-blue:0, orders-blue:1

streamPrefixм—җлҠ” Redis Cluster hash tag braceлҘј нҸ¬н•Ён•  мҲҳ м—ҶлӢӨ. Shard keyлҠ” hash tagлҘј м“°м§Җ м•Ҡм•„ Redis Clusterк°Җ hash slot кё°мӨҖмңјлЎң л¶„мӮ°н•  мҲҳ мһҲкІҢ н•ңлӢӨ.

Producer Routing

ProducerлҠ” лЎңм»¬ shard countлҘј source of truthлЎң м“°м§Җ м•ҠлҠ”лӢӨ. ProducerлҠ” coordinatorм—җм„ң routing metadataлҘј к°Җм ёмҳЁлӢӨ.

{
  "streamPrefix": "orders",
  "consumerGroup": "orders-consumer",
  "metadataVersion": 12,
  "shardCount": 8,
  "streamKeyPattern": "orders:{shardIndex}"
}

ProducerлҠ” мқҙ metadataлҘј мәҗмӢңн•ҳкі  partition keyлҘј [0, shardCount) лІ”мң„мқҳ shard indexлЎң лқјмҡ°нҢ…н•ңлӢӨ. metadataVersionмқҙ л°”лҖҢкұ°лӮҳ, cache TTLмқҙ л§ҢлЈҢлҗҳкұ°лӮҳ, publish мӨ‘ stale routingмқ„ к°җм§Җн•ҳл©ҙ cacheлҘј к°ұмӢ н•ңлӢӨ.

RoutingмқҖ лҸҷмқј metadata snapshot м•Ҳм—җм„ңл§Ң deterministicн•ҳлӢӨ.

  • к°ҷмқҖ routing protocol
  • к°ҷмқҖ shardCount
  • к°ҷмқҖ partition key

Shard countк°Җ л°”лҖҢл©ҙ лҸҷмқј partition keyлҸ„ лӢӨлҘё Redis Stream shardлЎң л“Өм–ҙк°Ҳ мҲҳ мһҲлӢӨ. CoordinatorлҠ” лӘЁл“  shardм—җ кұём№ң лҸҷмқј event idмқҳ global deduplicationмқ„ м ңкіөн•ҳм§Җ м•ҠлҠ”лӢӨ.

Duplicate Publish Boundary

мӨ‘мҡ” мӢңлӮҳлҰ¬мҳӨ:

  1. Producer Aк°Җ eventId=E, partitionKey=order-123мқ„ old routing metadataлЎң shard 2м—җ publishн•ңлӢӨ.
  2. Producerк°Җ м„ұкіө мқ‘лӢөмқ„ ліҙкё° м „м—җ Redis responseк°Җ мң мӢӨлҗңлӢӨ.
  3. мҡҙмҳҒмһҗк°Җ groupмқ„ 4 shardsм—җм„ң 8 shardsлЎң scaleн•ңлӢӨ.
  4. Producer Aк°Җ routing metadataлҘј к°ұмӢ н•ң л’Ө retryн•ңлӢӨ.
  5. к°ҷмқҖ partition keyк°Җ shard 6мңјлЎң лқјмҡ°нҢ…лҗ  мҲҳ мһҲлӢӨ.
  6. к°ҷмқҖ event IDк°Җ л‘җ shard мң„м№ҳм—җ мЎҙмһ¬н•  мҲҳ мһҲлӢӨ.

мҡҙмҳҒ м ңм•Ҫ:

  • Duplicate-sensitive workloadлҠ” shard scale-out/in мӨ‘ produceн•ҳл©ҙ м•Ҳ лҗңлӢӨ.
  • Scale м „м—җ producerлҘј мӨ‘м§Җн•ңлӢӨ.
  • In-flight XADDмҷҖ retry windowлҘј drainн•ңлӢӨ.
  • Scaleмқ„ мӢӨн–үн•ңлӢӨ.
  • Producer routing metadataлҘј к°ұмӢ н•ңлӢӨ.
  • Publishingмқ„ мһ¬к°ңн•ңлӢӨ.

Producer trafficмқ„ л©Ҳм¶ң мҲҳ м—ҶлӢӨл©ҙ workloadлҠ” at-least-onceлЎң м·Ёкёүн•ҳкі  application idempotencyлЎң ліҙнҳён•ҙм•ј н•ңлӢӨ.

Resharding Flow

Shard count ліҖкІҪмқҖ Coordinator Admin APIлЎңл§Ң мҲҳн–үн•ңлӢӨ.

sequenceDiagram
    participant Admin
    participant C as Coordinator
    participant Store as Metadata Store
    participant Stream as Redis Stream
    participant M as Member

    Admin->>C: Request resharding(targetShardCount)
    C->>Store: Acquire critical section
    C->>Store: Read group metadata
    C->>Store: Validate no active resharding
    C->>Store: Create PREPARING resharding state
    C->>Stream: Provision shard stream keys and consumer groups
    C->>Store: Activate new shard count
    C->>Store: Increment group epoch and recalculate target assignment
    C-->>Admin: Return reshardingId and old/new shard counts
    M->>C: Next heartbeat with current ownership/progress
    C-->>M: Revoke or assign response until target ownership converges

Member startupмқҖ лӢӨмқҢмңјлЎң м ңн•ңлҗңлӢӨ.

  • heartbeatлҘј нҶөн•ҙ coordinator metadata нҷ•мқё
  • member ID мғқм„ұ лҳҗлҠ” лЎңл“ң
  • {streamPrefix, consumerGroup}лЎң heartbeat мӢңмһ‘
  • heartbeat responseмқҳ assignment м Ғмҡ©

Member startupмқҖ лӢӨмқҢмқ„ н•ҳм§Җ м•ҠлҠ”лӢӨ.

  • local YAML shard countлҘј desired stateлЎң м ңм¶ң
  • group metadata мғқм„ұ лҳҗлҠ” ліҖкІҪ
  • consumer deployment лҳҗлҠ” listener concurrency ліҖкІҪ

Admin API Source of Truth

мҙҲкё° group мғқм„ұкіј shard scale-out/inмқҖ Coordinator Admin APIлЎңл§Ң мҲҳн–үн•ңлӢӨ.

  • shard count: coordinator group metadata
  • consumer concurrency: @StreamListener(concurrency = N)мқҙ л§Ңл“ңлҠ” consumer-side logical member мҲҳ

мІҳлҰ¬ мҲңм„ң:

  1. groupмқҙ мқҙлҜё мЎҙмһ¬н•ҳм§Җ м•ҠлҠ”м§Җ нҷ•мқён•ңлӢӨ.
  2. мҡ”мІӯ shard countлҘј кІҖмҰқн•ңлӢӨ.
  3. provisioningмқҙ мјңм ё мһҲмңјл©ҙ shard stream keyмҷҖ Redis consumer groupмқ„ мғқм„ұн•ңлӢӨ.
  4. shardCountмҷҖ groupEpoch=1мқ„ м ҖмһҘн•ңлӢӨ.
  5. duplicate create requestлҠ” 409 ConflictлЎң кұ°м Ҳн•ңлӢӨ.

Scale Out / Scale In

scale-outкіј scale-inмқҖ к°ҷмқҖ targetShardCount requestлЎң н‘ңнҳ„н•ңлӢӨ.

CoordinatorлҠ” лӢӨмқҢ мЎ°кұҙм—җм„ңл§Ң мҡ”мІӯмқ„ мҲҳлқҪн•ңлӢӨ.

  • groupм—җ active reshardingмқҙ м—ҶлӢӨ.
  • targetShardCountк°Җ нҳ„мһ¬ shard countмҷҖ лӢӨлҘҙлӢӨ.
  • targetShardCountк°Җ 0 мқҙмғҒмқҙлӢӨ.

Consumer parallelismмқҖ scale metadataк°Җ м•„лӢҲлӢӨ. мҡҙмҳҒмһҗлҠ” consumer deployment лҳҗлҠ” listener configurationмқ„ л°”кҝ” лі‘л ¬м„ұмқ„ мЎ°м •н•ңлӢӨ. coordinatorлҠ” heartbeatлЎң л“Өм–ҙмҳЁ logical memberл“Өмқ„ кҙҖм°°н•ҳкі  live member count кё°мӨҖмңјлЎң мһ¬л°°м№ҳн•ңлӢӨ.

Scale-inмқҖ м ңкұ° лҢҖмғҒ shard streamмқҳ л©”мӢңм§ҖлҘј лӢӨлҘё shardлЎң мқҙлҸҷн•ҳм§Җ м•ҠлҠ”лӢӨ. м ңкұ° лҢҖмғҒ shardлҠ” drain phaseм—җ л“Өм–ҙк°„лӢӨ.

  • targetShardCount=0мқҖ мң нҡЁн•ң full drainмқҙлӢӨ. лӘЁл“  м ңкұ° лҢҖмғҒ shardк°Җ drainлҗң л’Ө producer routingм—җм„ң лӘЁл“  shardлҘј м ңкұ°н•ңлӢӨ.
  • ProducerлҠ” routing metadataлҘј к°ұмӢ н•ҳкі  м ңкұ° лҢҖмғҒ shard indexлЎң мғҲ recordлҘј м“°м§Җ м•ҠлҠ”лӢӨ.
  • ConsumerлҠ” м ңкұ° лҢҖмғҒ shard streamм—җ мқҙлҜё мЎҙмһ¬н•ҳлҠ” recordлҘј кі„мҶҚ мІҳлҰ¬н•ңлӢӨ.
  • CoordinatorлҠ” live memberк°Җ м ңкұ° лҢҖмғҒ shardлҘј м•„м§Ғ мҶҢмң н•ҳкұ°лӮҳ revoke мӨ‘мқҙл©ҙ removed shardлҘј deprecated мІҳлҰ¬н•ҳл©ҙ м•Ҳ лҗңлӢӨ.
  • Scale-in лҸ„мӨ‘ live memberк°Җ лӘЁл‘җ expireлҗҳл©ҙ revoke ackлҘј ліҙлӮј heartbeat лҢҖмғҒмқҙ м—ҶлӢӨ. мқҙ кІҪмҡ° coordinatorлҠ” consumer-level revoke waitлҘј мғқлһөн•ҳкі  Redis-level drain checkлЎң м§„н–үн•ңлӢӨ.
  • CoordinatorлҠ” м ңкұ° лҢҖмғҒ physical stream shardл§ҲлӢӨ Redis XINFO GROUPSлҘј мЎ°нҡҢн•ҳкі , н•ҙлӢ№ streamмқ„ consumeн•ҳлҠ” лӘЁл“  Redis consumer groupмқҙ pending=0кіј known lag=0мқ„ ліҙкі н•  л•Ңк№Ңм§Җ кё°лӢӨл Өм•ј н•ңлӢӨ.
  • Redisк°Җ м ңкұ° лҢҖмғҒ shardмқҳ м–ҙл–Ө groupм—җ лҢҖн•ҙ lag=nullмқ„ л°ҳнҷҳн•ҳл©ҙ drain мҷ„лЈҢк°Җ мҰқлӘ…лҗҳм§Җ м•ҠмқҖ мғҒнғңмқҙлҜҖлЎң scale-inмқҖ DRAININGм—җ лЁёл¬ёлӢӨ.

Rollback

RollbackмқҖ metadataлЎң м•Ҳм „н•ҳкІҢ лҗҳлҸҢлҰҙ мҲҳ мһҲлҠ” resharding stateм—җм„ңл§Ң н—Ҳмҡ©н•ңлӢӨ. мғҲлЎң м¶”к°Җлҗң shard indexм—җ мқҙлҜё messageк°Җ м“°мҳҖлӢӨл©ҙ operator-defined policyлЎң drain, replay, ліҙм •н•ҙм•ј н•ңлӢӨ. RollbackмқҖ Redis Stream dataмҷҖ coordinator metadataлҘј н•ҳлӮҳмқҳ global transactionмңјлЎң л¬¶м§Җ м•ҠлҠ”лӢӨ.
