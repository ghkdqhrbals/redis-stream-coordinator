Metadata Version Correction

Clientм—җкІҢ л°ҳнҷҳлҗң metadata versionмқҖ мқҙнӣ„ Redisм—җм„ң мӮ¬лқјм§Ҳ мҲҳ мһҲлӢӨ. мҳҲлҘј л“Өм–ҙ coordinatorк°Җ metadata version 10мқ„ Redisм—җ м“°кі  heartbeat responseлЎң л°ҳнҷҳн–ҲлҠ”лҚ°, м§Ғнӣ„ Redis failover лҳҗлҠ” restoreлЎң version 9 мғҒнғңлЎң лҸҢм•„к°Ҳ мҲҳ мһҲлӢӨ. мқҙ кІҪмҡ° clientлҠ” Redisк°Җ нҳ„мһ¬ м ҖмһҘн•ң к°’ліҙлӢӨ лҚ” мөңмӢ  stateлҘј мқҙлҜё кҙҖмёЎн•ң кІғмқҙлӢӨ.

RedisлҘј source of truthлЎң мң м§Җн•ңлӢӨ. лҚ” лҶ’мқҖ client versionмқҖ Redis stateлҘј ліөкө¬н•  к¶Ңн•ңмқҙ м•„лӢҲлқј, clientк°Җ stale local viewлҘј л“Өкі  мһҲмқ„ мҲҳ мһҲлӢӨлҠ” мӢ нҳёлЎң м·Ёкёүн•ңлӢӨ.

  • Coordinator responseлҠ” нҳ„мһ¬ metadataVersion, assignmentEpoch, кҙҖл Ё member epochлҘј нҸ¬н•Ён•ңлӢӨ.
  • ConsumerмҷҖ producerлҠ” нҳ„мһ¬ process lifetime лҸҷм•Ҳ кҙҖмёЎн•ң мөңкі  versionмқ„ кё°м–өн•ңлӢӨ. лҚ” к°•н•ң rollback detectionмқҙ н•„мҡ”н•ң deploymentлҠ” module SPIлҘј нҶөн•ҙ мһ‘мқҖ local лҳҗлҠ” external persistent version storeлҘј м ңкіөн•ҙм•ј н•ңлӢӨ.
  • лӘЁл“  heartbeatмҷҖ routing refreshлҠ” clientк°Җ кҙҖмёЎн•ң мөңкі  versionмқ„ нҸ¬н•Ён•ңлӢӨ.
  • Coordinatorк°Җ clientMaxObservedVersion > storedMetadataVersionмқ„ ліҙл©ҙ metadata regressionмңјлЎң ліҙкі  metadata correction roundлҘј мӢңмһ‘н•ңлӢӨ. лӢЁ, coordinatorк°Җ metadata schemaлӮҳ coordination versionмқ„ н•ҙм„қн•  мҲҳ м—Ҷмңјл©ҙ fail closedн•ңлӢӨ.
  • CoordinatorлҠ” rollbackлҗң Redis versionмқ„ лӢЁмҲңнһҲ мҰқк°ҖмӢңмјң л¬ём ңлҘј "ліөкө¬"н•ҳл©ҙ м•Ҳ лҗңлӢӨ. мң мӢӨлҗң state transitionмқҳ лӮҙмҡ©мқҙ л¬ҙм—Үмқём§Җ м•Ң мҲҳ м—Ҷкё° л•Ңл¬ёмқҙлӢӨ.
  • request.metadataVersion > group.metadataVersionмқҙл©ҙ coordinatorлҠ” metadata correction roundлҘј мӢңмһ‘н•ҳкі  SYNC_METADATAлҘј л°ҳнҷҳн•ңлӢӨ.
  • SYNC_METADATAлҠ” нҳ„мһ¬ Redis metadataVersion, assignmentEpoch, кі„мҶҚ мң м§Җ к°ҖлҠҘн•ң assigned shard, pending target shardлҘј нҸ¬н•Ён•ңлӢӨ.
  • ConsumerлҠ” responseлҘј нҳ„мһ¬ metadata truthлЎң м Ғмҡ©н•ҳкі , мқҙлҜё мқҪкі  мһҲлҚҳ shard мӨ‘ assignedShardsм—җ лӮЁмқҖ shardл§Ң мң м§Җн•ңлӢӨ. лӮҳлЁём§ҖлҠ” мӢ к·ң readлҘј мӨ‘лӢЁн•ҳкі  revoke/drainн•ңлӢӨ.
  • Correction roundк°Җ нҷңм„ұнҷ”лҗң лҸҷм•Ҳ coordinatorлҠ” нҳ„мһ¬ Redis metadata versionкіј лӢӨлҘё heartbeatлҘј ліҙлӮҙлҠ” лӘЁл“  known consumerм—җкІҢ SYNC_METADATAлҘј л°ҳнҷҳн•ңлӢӨ.
  • Consumerк°Җ нҳ„мһ¬ metadata versionмңјлЎң heartbeatн–Ҳм§Җл§Ң local revoke/drain лҳҗлҠ” лӢӨлҘё memberмқҳ revoke л•Ңл¬ём—җ target shardк°Җ м•„м§Ғ л§үнҳҖ мһҲмңјл©ҙ coordinatorлҠ” REVOKE_PENDINGмқ„ л°ҳнҷҳн•ңлӢӨ.
  • REVOKE_PENDINGлҸ„ л©ұл“ұм ҒмқҙлӢӨ. ConsumerлҠ” drainмқ„ кі„мҶҚн•ҳкі , лӮҳмӨ‘м—җ OKлҘј л°ӣмқ„ л•Ңк№Ңм§Җ мӢ к·ң shard readлҘј мӢңмһ‘н•ҳм§Җ м•ҠлҠ”лӢӨ.
  • мӢ к·ң shard readлҠ” OKм—җм„ңл§Ң мӢңмһ‘н•ңлӢӨ.
  • CoordinatorлҠ” consumer reportлЎң Redis metadataлҘј мһ¬кө¬м„ұн•ҳм§Җ м•Ҡкі , client versionм—җ л§һм¶”л Өкі  Redis versionл§Ң мҰқк°ҖмӢңнӮӨм§ҖлҸ„ м•ҠлҠ”лӢӨ.

Rebalance Drain Failure Scenario

к°ҖмһҘ мӨ‘мҡ”н•ң rebalance flowлҠ” лӢӨмқҢмқҙлӢӨ.

member A owns shard S
member C joins or capacity changes
coordinator computes target owner B/C for shard S
coordinator asks member A to drain shard S through heartbeat response
member A stops new reads, drains in-flight work, then reports REVOKED
coordinator assigns shard S to the new target only after release is accepted

DRAIN instructionмқҖ нңҳл°ңм„ұ messageк°Җ м•„лӢҲлӢӨ. DRAINмқҖ Redisм—җ кё°лЎқлҗң metadataм—җм„ң нҢҢмғқлҗң heartbeat responseмқҙлӢӨ.

  • target assignmentлҠ” shard Sк°Җ лҚ” мқҙмғҒ member A мҶҢмң к°Җ м•„лӢҲлқјкі  л§җн•ңлӢӨ.
  • current assignmentлҠ” member AлҘј current лҳҗлҠ” revoking ownerлЎң кё°лЎқн•ңлӢӨ.
  • member B лҳҗлҠ” CлҠ” releaseк°Җ мҲҳлқҪлҗ  л•Ңк№Ңм§Җ shard SлҘј pendingмңјлЎңл§Ң л°ӣлҠ”лӢӨ.
  • assignment epochк°Җ мқҙ нҢҗлӢЁмқ„ мӢқлі„н•ңлӢӨ.
