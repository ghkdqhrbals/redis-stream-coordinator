RedisStream Spring Boot Starter and Integration Contract

лӘ©н‘ң

com.redisstream:redisstream-spring-boot-starterлҠ” м• н”ҢлҰ¬мјҖмқҙм…ҳмқҙ Redis Stream Coordinatorм—җ м—°кІ°н•ҳкё° мң„н•ҙ н•„мҡ”н•ң лҹ°нғҖмһ„ кө¬м„ұмҡ”мҶҢлҘј м ңкіөн•ңлӢӨ.

Coordinator м„ңлІ„лҠ” control planeмқҙлӢӨ. м• н”ҢлҰ¬мјҖмқҙм…ҳмқҖ starterлҘј нҶөн•ҙ coordinator groupм—җ joinн•ҳкі , heartbeatлҘј ліҙлӮҙкі , shard assignmentлҘј л°ӣкі , local shard workerлҘј мӢңмһ‘н•ҳкұ°лӮҳ мӨ‘м§Җн•ҳл©°, revoke/drain progressлҘј ліҙкі н•ңлӢӨ.

StarterлҠ” нҠ№м • Redis Stream мІҳлҰ¬ н”„л Ҳмһ„мӣҢнҒ¬лҘј к°•м ңн•ҳм§Җ м•ҠлҠ”лӢӨ. м• н”ҢлҰ¬мјҖмқҙм…ҳмқҖ shard lifecycle callbackмқ„ м§Ғм ‘ кө¬нҳ„н•  мҲҳ мһҲкі , мӣҗн•ҳл©ҙ built-in Redis Stream polling adapterлҘј мӮ¬мҡ©н•  мҲҳ мһҲлӢӨ. л№„мҰҲлӢҲмҠӨ handler мӢӨн–ү, retry, DLQ, idempotency, transaction boundaryлҠ” м• н”ҢлҰ¬мјҖмқҙм…ҳмқҙ мҶҢмң н•ңлӢӨ.

кіөк°ң нҶөн•© API

м• н”ҢлҰ¬мјҖмқҙм…ҳ лӘЁл“ҲмқҖ Spring Boot starter packageл§Ң мқҳмЎҙн•ҳл©ҙ лҗңлӢӨ. Coordinator server лӮҙл¶Җмқҳ mutex guard, aspect, critical-section annotationмқҖ кө¬нҳ„ л””н…Ңмқјмқҙл©° кіөк°ң APIк°Җ м•„лӢҲлӢӨ.

м• н”ҢлҰ¬мјҖмқҙм…ҳмқҙ м§Ғм ‘ мӮ¬мҡ©н•ҳлҠ” мЈјмҡ” APIлҠ” лӢӨмқҢ м •лҸ„лЎң м ңн•ңн•ңлӢӨ.

API м—ӯн• 
@StreamConfiguration coordinator-managed listener methodк°Җ кіөмң н•  polling м„Өм •кіј poller thread pool size кё°ліёк°’мқ„ м„ м–ён•ңлӢӨ.
@StreamListener business handler methodлҘј listener endpointлЎң н‘ңмӢңн•ҳкі  stream prefix, group ID, startup, concurrencyлҘј endpoint лӢЁмң„лЎң м„Өм •н•ңлӢӨ.
CoordinatorConsumerProperties.consumer(...) н•ҳлӮҳмқҳ {streamPrefix, consumerGroupName}м—җ лҢҖн•ң managed consumerлҘј л“ұлЎқн•ңлӢӨ.
ProducerRoutingProperties.producer(...) н•ҳлӮҳмқҳ {streamPrefix, consumerGroupName}м—җ лҢҖн•ң producer routing cacheлҘј л“ұлЎқн•ңлӢӨ.
RedisStreamPublisher.publish(...) partition keyлҘј active Redis Stream shardлЎң routingн•ҳкі  recordлҘј XADDн•ңлӢӨ.
CoordinatorShardLifecycle м• н”ҢлҰ¬мјҖмқҙм…ҳмқҙ м§Ғм ‘ workerлҘј мҡҙмҳҒн•  л•Ң shard assign/revoke callbackмқ„ л°ӣлҠ”лӢӨ.
RedisStreamMessageHandler built-in polling adapterк°Җ business handlerлҘј нҳём¶ңн•ңлӢӨ. ACK, ACKDEL, NACKмқҖ м• н”ҢлҰ¬мјҖмқҙм…ҳ мҪ”л“ңк°Җ лӘ…мӢңм ҒмңјлЎң мҲҳн–үн•ңлӢӨ.

мөңмҶҢ annotation кё°л°ҳ consumer л“ұлЎқ мҳҲмӢңлҠ” лӢӨмқҢкіј к°ҷлӢӨ.

@StreamConfiguration
class OrdersConsumer {
    @StreamListener(
        id = "orders-consumer-a",
        streamPrefix = "orders",
        groupId = "orders-consumer",
        concurrency = "4",
    )
    fun consume(message: ConsumedRedisStreamMessage) {
        // Business processing.
        message.ack()
    }
}

Annotation listenerм—җм„ң concurrency = "4"лҠ” к°ҷмқҖ м• н”ҢлҰ¬мјҖмқҙм…ҳ н”„лЎңм„ёмҠӨ м•Ҳм—җ 4к°ңмқҳ л…јлҰ¬ coordinator memberлҘј л§Ңл“ лӢӨ. к°Ғ л…јлҰ¬ memberлҠ” к°ңлі„ memberId, heartbeat loop, Redis consumer name, assignment state, shard ownershipмқ„ к°Җм§„лӢӨ. мқҙ л°©мӢқмқҙ KafkaмҷҖ мң мӮ¬н•ң listener concurrency лӘЁлҚёмқҙлӢӨ.

Annotation listenerлҠ” н•ӯмғҒ мқҙ logical-member split лӘЁлҚёмқ„ мӮ¬мҡ©н•ңлӢӨ. StarterлҠ” split toggleмқ„ кіөк°ң м„Өм •мңјлЎң м ңкіөн•ҳм§Җ м•Ҡмңјл©°, concurrency = "4"к°Җ н•ҳлӮҳмқҳ coordinator member лӮҙл¶Җ local poller л„Ө к°ңлҘј мқҳлҜён•ҳлҠ” лӘЁл“ңлҠ” м§Җмӣҗн•ҳм§Җ м•ҠлҠ”лӢӨ.

Shard ownershipкіј member concurrencyлҠ” лі„к°ңмқҙлӢӨ. CoordinatorлҠ” к°Ғ shardлҘј м •нҷ•нһҲ н•ҳлӮҳмқҳ live memberм—җл§Ң л°°м№ҳн•ҳм§Җл§Ң, н•ҳлӮҳмқҳ memberк°Җ м—¬лҹ¬ shardлҘј мҶҢмң н•  мҲҳ мһҲлӢӨ. Listener concurrencyлҠ” assignmentм—җ м°ём—¬н•ҳлҠ” л…јлҰ¬ member мҲҳлҘј лҠҳлҰ¬лҠ” кё°лҠҘмқҙл©°, shard н•ҳлӮҳлҠ” live owner н•ҳлӮҳл§Ң к°Җм§„лӢӨлҠ” к·ңм№ҷмқҖ л°”кҫём§Җ м•ҠлҠ”лӢӨ.

Annotation кё°л°ҳ consumerм—җм„ңлҠ” starterк°Җ runtime memberIdлҘј мһҗлҸҷ мғқм„ұн•ңлӢӨ. idлҠ” listener endpoint мӢқлі„мһҗмқҙл©° coordinator member IDк°Җ м•„лӢҲлӢӨ.

Heartbeat intervalкіј rebalance timeoutмқҖ shared coordination version timing кё°ліёк°’мқ„ мӮ¬мҡ©н•ңлӢӨ. rebalanceTimeoutмқҖ revokeлҗң shardлҘј drainн•  мҲҳ мһҲлҸ„лЎқ coordinatorк°Җ н•ҙлӢ№ memberлҘј кё°лӢӨлҰ¬лҠ” мөңлҢҖ мӢңк°„мқҙлӢӨ. мқҙ мӢңк°„ м•Ҳм—җ revoke мҷ„лЈҢ ліҙкі к°Җ мҳӨм§Җ м•Ҡмңјл©ҙ coordinatorлҠ” memberлҘј fenceн•ҳкі  shardлҘј мһ¬н• лӢ№н•  мҲҳ мһҲлӢӨ.

Bean кё°л°ҳ consumer/producer л“ұлЎқ мҳҲмӢңлҠ” лӢӨмқҢкіј к°ҷлӢӨ.

@Bean
fun ordersConsumer(): CoordinatorConsumerProperties =
    CoordinatorConsumerProperties.consumer("orders", "orders-consumer") {
        runtimeMaxConcurrency = 4
    }

@Bean
fun ordersProducer(): ProducerRoutingProperties =
    ProducerRoutingProperties.producer("orders", "orders-consumer") {
        xadd.maxLen = 100_000
    }

Bean кё°л°ҳ кІҪлЎңм—җм„ң runtimeMaxConcurrency = 4лҠ” н•ҳлӮҳмқҳ coordinator memberм—җ лҢҖн•ң local worker capacityмқҙлӢӨ. л„Ө к°ңмқҳ member IDлҘј л§Ңл“Өм§ҖлҠ” м•ҠлҠ”лӢӨ. KafkaмҷҖ мң мӮ¬н•ң member fan-outмқҙ н•„мҡ”н•ҳл©ҙ @StreamListener(concurrency = "N")мқ„ мӮ¬мҡ©н•ҳкұ°лӮҳ м„ңлЎң лӢӨлҘё member identityлҘј к°Җм§„ managed consumerлҘј лӘ…мӢңм ҒмңјлЎң м—¬лҹ¬ к°ң л“ұлЎқн•ҙм•ј н•ңлӢӨ.

мөңмҶҢ publish мҳҲмӢңлҠ” лӢӨмқҢкіј к°ҷлӢӨ.

redisStreamPublisher.publish(
    partitionKey = "order-123",
    fields = mapOf("eventId" to "evt-1", "payload" to payload),
)

Public YAML Contract

кіөмӢқм ҒмңјлЎң м ңкіөн•ҳлҠ” YAML м„Өм •мқҖ coordinator endpoint н•ҳлӮҳл§ҢмқҙлӢӨ.

redis-stream-coordinator:
  coordinator-base-url: http://localhost:8080

Consumer/producerмқҳ stream prefix, consumer group name, polling size, timeout, ACK м •мұ…, producer routing refresh, XADD MAXLEN к°ҷмқҖ runtime м„Өм •мқҖ мҪ”л“ңм—җм„ң beanмңјлЎң м •мқҳн•ңлӢӨ. мқҙл ҮкІҢ н•ҙм•ј н•ң м• н”ҢлҰ¬мјҖмқҙм…ҳ м•Ҳм—җм„ң м—¬лҹ¬ consumer/producerлҘј лӘ…мӢңм ҒмңјлЎң л§Ңл“Ө мҲҳ мһҲкі , м• н”ҢлҰ¬мјҖмқҙм…ҳ кі мң  м„Өм • мӢңмҠӨн…ңмқҙлӮҳ лІ„м „ м •мұ…кіј м¶©лҸҢн•ҳм§Җ м•ҠлҠ”лӢӨ.

Consumer м„Өм •

consumerGroupNameмқҖ Redis Stream consumer groupмқҳ л…јлҰ¬ мқҙлҰ„мқҙлӢӨ. StarterлҠ” member-nameмқ„ public YAML м„Өм •мңјлЎң м ңкіөн•ҳм§Җ м•ҠлҠ”лӢӨ. Heartbeat protocol compatibilityм—җ н•„мҡ”н•ң member display fieldлҠ” лӮҙл¶Җм ҒмңјлЎң consumerGroupNameм—җм„ң нҢҢмғқн•ңлӢӨ.

Managed consumer beanмқҙ мғқм„ұлҗ  л•Ң starterлҠ” м„Өм •лҗң streamPrefixмҷҖ consumerGroupNameм—җ лҢҖн•ң coordinator routing metadataлҘј лЁјм Җ мЎ°нҡҢн•ңлӢӨ. Groupмқҙ м—Ҷкұ°лӮҳ, shardCountк°Җ 0мқҙкұ°лӮҳ, active shard metadataк°Җ л¶Ҳмҷ„м „н•ҳл©ҙ м• н”ҢлҰ¬мјҖмқҙм…ҳ мӢңмһ‘ мӢңм җм—җ мӢӨнҢЁн•ңлӢӨ.

@Configuration(proxyBeanMethods = false)
class OrdersConsumerConfiguration {
    @Bean
    fun ordersConsumerProperties(): CoordinatorConsumerProperties =
        CoordinatorConsumerProperties.consumer(
            streamPrefix = "orders",
            consumerGroupName = "orders-consumer",
        ) {
            runtimeMaxConcurrency = 4
            heartbeatInterval = Duration.ofSeconds(3)
            rebalanceTimeout = Duration.ofSeconds(60)
            gracefulLeaveOnStop = true
        }
}

Starterк°Җ лӢҙлӢ№н•ҳлҠ” мҳҒм—ӯ:

  • heartbeat scheduling
  • coordinator HTTP call
  • memberEpochмҷҖ metadataVersion tracking
  • ownedShardsмҷҖ revokingShards reporting
  • assignment diffing
  • assign, pending, revoke, fenced callback нҳём¶ң
  • runtime capacity reporting
  • built-in polling мӮ¬мҡ© мӢң shard progress reporting

Heartbeat status мІҳлҰ¬:

Status Starter лҸҷмһ‘
OK AssignmentлҘј м Ғмҡ©н•ҳкі  мӢ к·ң assigned shard workerлҘј мӢңмһ‘н•ңлӢӨ.
SYNC_METADATA Local metadata versionмқ„ к°ұмӢ н•ҳкі , нҳ„мһ¬ owned shard мӨ‘ coordinatorк°Җ кі„мҶҚ assignedлЎң н‘ңмӢңн•ң shardл§Ң мң м§Җн•ңлӢӨ. лӮҳлЁём§ҖлҠ” revoke/drainн•ҳл©° мӢ к·ң shardлҠ” мӢңмһ‘н•ҳм§Җ м•ҠлҠ”лӢӨ.
REVOKE_PENDING Revoke/drainмқ„ кі„мҶҚн•ҳкі , мқҙнӣ„ OKк°Җ мҳ¬ л•Ңк№Ңм§Җ мӢ к·ң shardлҘј pending мғҒнғңлЎң л‘”лӢӨ.

  • payload deserialization
  • л№„мҰҲлӢҲмҠӨ handler transaction
  • мҷёл¶Җ DB/API side effect
  • DLQмҷҖ retry policy

Shard Lifecycle API

м• н”ҢлҰ¬мјҖмқҙм…ҳмқҖ CoordinatorShardLifecycleмқ„ кө¬нҳ„н•ңлӢӨ.

@Component
class OrdersShardLifecycle : CoordinatorShardLifecycle {
    override fun onAssigned(shards: Set<CoordinatorShard>, context: CoordinatorConsumerContext) {
        // assigned shard workerлҘј мӢңмһ‘н•ңлӢӨ.
    }

    override fun onRevoked(
        shards: Set<CoordinatorShard>,
        context: CoordinatorConsumerContext,
    ): Set<CoordinatorShard> {
        // мғҲ readлҘј л©Ҳм¶”кі  in-flight workлҘј drainн•ң л’Ө мҷ„лЈҢлҗң shardлҘј л°ҳнҷҳн•ңлӢӨ.
        return shards
    }
}

Lifecycle callbackмқҖ idempotentн•ҙм•ј н•ңлӢӨ. reconnect, retry, heartbeat мӨ‘ліөмңјлЎң к°ҷмқҖ assignmentк°Җ л‘җ лІҲ мқҙмғҒ м „лӢ¬лҗ  мҲҳ мһҲлӢӨ.

Built-in Redis Stream Polling

Starterк°Җ Redis Stream readлҘј мҲҳн–үн•ҳкІҢ н•ҳл Өл©ҙ RedisStreamMessageHandler beanмқ„ м ңкіөн•ҳкі , polling мҳөм…ҳмқ„ мҪ”л“ңм—җм„ң м„Өм •н•ңлӢӨ.

@Configuration(proxyBeanMethods = false)
class OrdersPollingConfiguration {
    @Bean
    fun ordersConsumerProperties(): CoordinatorConsumerProperties =
        CoordinatorConsumerProperties.consumer(
            streamPrefix = "orders",
            consumerGroupName = "orders-consumer",
        ) {
            redis.pollBatchSize = 10
            redis.pollTimeout = Duration.ofSeconds(1)
        }
}

@Component
class OrdersMessageHandler : RedisStreamMessageHandler {
    override fun handle(message: ConsumedRedisStreamMessage) {
        // л№„мҰҲлӢҲмҠӨ мІҳлҰ¬к°Җ м„ұкіөн•ң л’Ө м• н”ҢлҰ¬мјҖмқҙм…ҳ мҪ”л“ңк°Җ м§Ғм ‘ ACKн•ңлӢӨ.
        message.ack()
    }
}

Built-in polling adapterлҠ” мһҗлҸҷ ACKлҘј мҲҳн–үн•ҳм§Җ м•ҠлҠ”лӢӨ. л№„мҰҲлӢҲмҠӨ side effectк°Җ м„ұкіөн•ң л’Ө message.ack()лҘј нҳём¶ңн•ҳкі , stream entryк№Ңм§Җ мӮӯм ңн•ҙм•ј н•ҳлҠ” кІҪмҡ° message.ackDel()мқ„ нҳём¶ңн•ңлӢӨ. Redis XNACKмқ„ мӮ¬мҡ©н•ҙм•ј н•ҳлҠ” кІҪмҡ°м—җлҠ” message.nack()мқ„ лӘ…мӢңм ҒмңјлЎң нҳём¶ңн•ңлӢӨ.

Polling semantics:

  • coordinator ownershipмқҖ shard лӢЁмң„мқҙл©°, н•ҳлӮҳмқҳ shardлҠ” мөңлҢҖ н•ҳлӮҳмқҳ live memberл§Ң мқҪмқ„ мҲҳ мһҲлӢӨ.
  • listener concurrencyлҠ” м• н”ҢлҰ¬мјҖмқҙм…ҳ н”„лЎңм„ёмҠӨ м•Ҳм—җ мғқм„ұн•  л…јлҰ¬ member мҲҳмқҙлӢӨ.
  • bean кё°л°ҳ runtimeMaxConcurrencyлҠ” н•ҳлӮҳмқҳ л…јлҰ¬ memberк°Җ к°Җм§„ local worker capacityмқҙлӢӨ.
  • н•ҳлӮҳмқҳ memberк°Җ м—¬лҹ¬ shardлҘј мҶҢмң н•ҳл©ҙ workerк°Җ owned shardлҘј round-robinмңјлЎң мҲңнҡҢн•ңлӢӨ.
  • л’ӨмӘҪ shard indexлҸ„ кі„мҶҚ poll/processingлҗҳм–ҙм•ј н•ҳл©°, н•ҳлӮҳмқҳ memberк°Җ worker мҲҳліҙлӢӨ л§ҺмқҖ shardлҘј мҶҢмң н–ҲлӢӨлҠ” мқҙмң лЎң мҳҒкө¬м Ғмқё shard starvationмқҙ л°ңмғқн•ҳл©ҙ м•Ҳ лҗңлӢӨ.
  • built-in adapterлҠ” shard-local orderingкіј мӨ‘ліө local read л°©м§ҖлҘј мң„н•ҙ shardлӢ№ н•ҳлӮҳмқҳ active pollл§Ң н—Ҳмҡ©н•ңлӢӨ.

Failed-message мІҳлҰ¬лҠ” redis.failure.modeлЎң лі„лҸ„ м„Өм •н•ңлӢӨ. кё°ліёк°’мқҖ LEAVE_PENDINGмқҙл©° classic Redis Stream retry behaviorлҘј мң м§Җн•ңлӢӨ. XNACKмқҖ лӘ…мӢңм ҒмңјлЎң м„Өм •н•ҳкі  Redis лІ„м „мқҙ м§Җмӣҗн•  л•Ңл§Ң мӮ¬мҡ©н•ңлӢӨ.

Consumer Progress Reporting

Built-in polling lifecycleмқҖ heartbeatм—җ shardлі„ progressлҘј нҸ¬н•Ён•  мҲҳ мһҲлӢӨ.

  • last delivered Redis Stream ID
  • last acknowledged Redis Stream ID
  • pending count
  • in-flight count
  • shard read state

CoordinatorлҠ” мқҙ к°’мқ„ monitoringкіј drain нҢҗлӢЁм—җ мӮ¬мҡ©н•ңлӢӨ. мқҙ к°’мқҖ exactly-once ліҙмһҘмқҙ м•„лӢҲлӢӨ.

Producer м„Өм •

ProducerлҸ„ YAMLм—җлҠ” shared coordinator endpointл§Ң л‘”лӢӨ. Routing identityмҷҖ producer лҸҷмһ‘мқҖ мҪ”л“ңм—җм„ң м„Өм •н•ңлӢӨ.

@Configuration(proxyBeanMethods = false)
class OrdersProducerConfiguration {
    @Bean
    fun ordersProducerProperties(): ProducerRoutingProperties =
        ProducerRoutingProperties.producer(
            streamPrefix = "orders",
            consumerGroupName = "orders-consumer",
        ) {
            routingRefreshInterval = Duration.ofSeconds(30)
            publishMaxAttempts = 1
            xadd.maxLen = 10_000_000
            xadd.approximateTrimming = true
        }
}

ProducerRoutingCache beanлҸ„ мғқм„ұ мӢң к°ҷмқҖ мҙҲкё° metadata validationмқ„ мҲҳн–үн•ҳкі  local routing cacheлҘј мұ„мҡҙлӢӨ. Prefix/group shard metadataк°Җ м—ҶлҠ” мғҒнғңлҠ” мІ« publish мӢңм җмқҳ м—җлҹ¬к°Җ м•„лӢҲлқј startup errorмқҙлӢӨ.

ProducerлҠ” heartbeatлҘј ліҙлӮҙм§Җ м•ҠлҠ”лӢӨ. Shard м¶”к°ҖмҷҖ shard count ліҖкІҪмқҖ мЈјкё°м Ғ routing metadata refreshлЎң producerм—җ м „нҢҢлҗңлӢӨ. routingRefreshIntervalмқҖ мқјл°ҳм Ғмқё м „нҢҢ м§Җм—° мғҒн•ңмқҙкі , routing cache leaseлҠ” coordinator refresh м—Ҷмқҙ producerк°Җ кі„мҶҚ publishн•  мҲҳ мһҲлҠ” мөңлҢҖ мӢңк°„мқҙлӢӨ. Cache leaseк°Җ л§ҢлЈҢлҗң л’Өм—җлҸ„ refreshк°Җ мӢӨнҢЁн•ҳл©ҙ stale routingмқ„ л¬ҙкё°н•ң мӮ¬мҡ©н•ҳм§Җ м•Ҡкі  publishлҘј fail closedн•ҙм•ј н•ңлӢӨ.

м• н”ҢлҰ¬мјҖмқҙм…ҳмқҖ RedisStreamPublisherлЎң publishн•ңлӢӨ.

redisStreamPublisher.publish(
    partitionKey = "order-123",
    fields = mapOf("eventId" to "evt-1", "payload" to "..."),
)

PublisherлҠ” coordinatorмқҳ producer routing metadataлҘј мқҪкі , shard countмҷҖ shard metadataлҘј кё°мӨҖмңјлЎң partition keyлҘј stream shardм—җ л§Өн•‘н•ң л’Ө XADDн•ңлӢӨ.

Producer pathлҠ” global event id deduplicationмқ„ м ңкіөн•ҳм§Җ м•ҠлҠ”лӢӨ. Shard scale-out/inмңјлЎң routing metadataк°Җ л°”лҖҢл©ҙ к°ҷмқҖ event idк°Җ м„ңлЎң лӢӨлҘё shardм—җ м“°мқј мҲҳ мһҲлӢӨ. мӨ‘ліөм—җ лҜјк°җн•ң workloadлҠ” scale мһ‘м—… лҸҷм•Ҳ producerлҘј pauseн•ҳкі  in-flight XADDмҷҖ retry windowлҘј drainн•ң л’Ө routing metadataлҘј refreshн•ҙм•ј н•ңлӢӨ.

Redis Version Compatibility

Processing Guarantee

Starterмқҳ кё°ліё ліҙмһҘмқҖ at-least-once processingмқҙлӢӨ.

мӨ‘ліө attemptлҠ” лӢӨмқҢ мғҒнҷ©м—җм„ң л°ңмғқн•  мҲҳ мһҲлӢӨ.

  • Redisк°Җ messageлҘј л°ҳнҷҳн•ң л’Ө processк°Җ XACK м „м—җ crash
  • pending entryлҘј лӢӨлҘё consumerк°Җ reclaim
  • Redis XADD кІ°кіјк°Җ л¶Ҳнҷ•мӢӨн•ң мғҒнғңм—җм„ң producer retry
  • shard scale-out/in мӨ‘ producer routing metadata ліҖкІҪ
  • handlerк°Җ мҷёл¶Җ DB/API side effect мқјл¶Җл§Ң м„ұкіөн•ң л’Ө мӢӨнҢЁ

StarterлҠ” single-processing guaranteeлҘј мЈјмһҘн•ҳм§Җ м•ҠлҠ”лӢӨ. мӨ‘ліө side effectлҘј н—Ҳмҡ©н•  мҲҳ м—ҶлҠ” м• н”ҢлҰ¬мјҖмқҙм…ҳмқҖ domain-level idempotency, deduplication, unique constraint, compensationмқ„ м§Ғм ‘ кө¬нҳ„н•ҙм•ј н•ңлӢӨ.

StarterлҠ” лӘЁл“Ҳмқҙ м •мқҳн•ң protocolVersionмқ„ heartbeat requestм—җ нҸ¬н•Ён•ңлӢӨ. мқҙ к°’мқҖ heartbeat м „мҡ© versionмқҙ м•„лӢҲлқј coordinator-module coordination versionмқҙлӢӨ. Coordinator serverлҸ„ лӘЁл“Ҳмқҙ м •мқҳн•ң supported coordination version rangeл§Ң н—Ҳмҡ©н•ңлӢӨ. м• н”ҢлҰ¬мјҖмқҙм…ҳмқҙлӮҳ мҡҙмҳҒ YAMLм—җм„ң coordination versionмқ„ м§Ғм ‘ м„Өм •н•ҳм§Җ м•ҠлҠ”лӢӨ.
